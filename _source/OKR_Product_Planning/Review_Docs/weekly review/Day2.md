# Day 2 Review (Model Updates & Business Service)

- **Critical** – `createBusinessFromSignup` treats `employee_count` as a strict number and defaults via `employee_count || 10`; JSON bodies deliver string values, so `"25"` is rejected and `"0"` silently becomes `10` (`server/services/BusinessCreationService.js:38`, `server/services/BusinessCreationService.js:40`, `server/services/BusinessCreationService.js:43`). Parse and normalise the input before validation to match API payloads.
- **High** – The new helper returns a different size bucket than the utility that actually seeds the document: `calculateSizeCategory` flags 50 employees as “small” while `BusinessCreationService.getSizeCategory` reports “medium” (`server/utils/businessDefaults.js:14`, `server/services/BusinessCreationService.js:175`). This inconsistency will leak into any UI/API that relies on the service method.
- **Critical** – Consultant signup still cannot work as designed because `business_id` remains required and `email` is globally unique; consultants need `business_id = null` and shared emails per tenant per the plan (`server/models/User.js:10`, `server/models/User.js:21`). Relax those constraints (e.g., make `business_id` optional for consultants and drop the single-field unique index) before exposing the multi-role signup.
- **High** – Invitation defaults use legacy role names (`'team_member'`, `'manager'`, etc.), but auto-account creation will push those values straight into the `User` model whose enum only accepts the new upper-case roles; the first untouched invitation will fail validation and never create an account (`server/models/Invitation.js:39`, `server/models/User.js:35`). Normalise incoming roles or align the enum and defaults.
- **Medium** – Exposing `template_id` via a getter alone is not enough: without enabling getters on toJSON/toObject the alias never appears in API payloads, so consumers will still have to read `assessment_template_id` (`server/models/Invitation.js:66`). Add `invitationSchema.set('toJSON', { getters: true })` (and/or mirror the field explicitly) to deliver the new contract.
