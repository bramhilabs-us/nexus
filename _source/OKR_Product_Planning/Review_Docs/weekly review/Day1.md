# Day 1 Review (Question Library & Template Models)

- **High** – Re-running the extraction script rewrites every question because `metadata.created_at` is stamped with `new Date()` on each execution; this makes the generated JSON non-deterministic and guarantees noisy diffs whenever the script is re-run (`scripts/extractQuestionsFromMockup.js:130`). Persist a fixed source timestamp or only set the field when absent.
- **High** – `AssessmentTemplate` never rejects duplicate question IDs. `selected_questions.length` is counted verbatim while the lookup collapses duplicates, so validation passes, `question_count` is inflated, and front-end/API consumers will see fewer questions than expected (`server/models/AssessmentTemplate.js:174`, `server/models/AssessmentTemplate.js:205`). Add a duplicate check (e.g., `new Set(allQuestionIds)`) before saving.
- **Medium** – The pre-save hook enforces a minimum total question count but does not ensure each dimension carries at least one question; a misconfigured template can leave, say, `strength` empty and still persist, which breaks downstream scoring assumptions (`server/models/AssessmentTemplate.js:230`). Either call `validateDimensions()` inside the hook or add explicit per-dimension checks.
