# Karvia Business – Code Review Notes

**Release Target:** 27 September 2024  
**Reviewer:** Codex (GPT-5)  
**Scope:** Architecture and integration review across main API, IAM engine, planner/cascade services, and deployment automation.

## Critical Issues

### 1. JWT Secret Inconsistency Across Services
- **Description:** IAM engine signs tokens using hard-coded fallbacks (`karvia-business-secret`, `karvia-refresh-secret`) when environment variables are absent, while the API gateway verifies tokens via central config (`config.get('security.jwt.secret')`). Tokens issued without explicitly set secrets are rejected everywhere else.
- **Risk:** Authentication outage in non-production environments; production vulnerability if defaults are ever deployed.  
- **Affected Files:** `engines/iam/index.js:109`, `engines/iam/index.js:115`, `server/middleware/auth.js:22`.  
- **Recommendation:** Remove hard-coded secrets and require a shared `JWT_SECRET`/`JWT_REFRESH_SECRET` supplied through configuration for all services.

### 2. Missing Engine Endpoint Configuration in Compose
- **Description:** `karvia-app` container receives no `*_ENGINE_URL`/`ENGINE_HOST` values. Proxy routes default to `http://localhost:808x`, which loop back to the API container instead of the engine container.
- **Risk:** All cross-service calls fail once deployed via Docker Compose.  
- **Affected File:** `docker-compose.yml:12-47` (environment block).  
- **Recommendation:** Inject the engine hostnames (e.g., `http://karvia-engines:8081`) via environment variables or refactor to use service discovery outputs.

## High Priority Issues

### 3. Engine Docker Image Cannot Build
- **Description:** `Dockerfile.engines` copies `engines/package*.json`, but no root-level package manifest exists. Build stops with “no source files were specified”.
- **Risk:** Release images for the engines cannot be produced.  
- **Affected File:** `Dockerfile.engines:18-21`.  
- **Recommendation:** Provide a root `engines/package.json` aggregating engine dependencies or adjust the Dockerfile to install per-engine packages after copying sources.

### 4. Startup Scripts Never Progress Past Dependency Checks
- **Description:** Both startup scripts poll MongoDB and Redis using `curl mongodb://...`/`curl redis://...`, protocols curl cannot speak, so waits never resolve.
- **Risk:** Containers hang indefinitely and the stack never becomes healthy.  
- **Affected Files:** `scripts/start.sh:16-27`, `scripts/start-engines.sh:13-24`.  
- **Recommendation:** Replace with suitable checks (`nc`, `mongosh`, `redis-cli`, or Docker healthcheck dependencies).

### 5. Cascade Engine Calls Non-existent Planner Endpoint
- **Description:** Cascade engine posts to `/api/cascade/generate-plan`, but planner service exposes only `/api/planning/*` routes.
- **Risk:** Automated cascade feature always fails (404/ECONNREFUSED).  
- **Affected Files:** `server/services/cascade-engine.js:191`, `engines/planner/index.js` (missing handler).  
- **Recommendation:** Implement corresponding planner endpoint or adjust cascade engine to available routes.

## Medium Priority Issues

### 6. Invalid Mongo Init Volume Reference
- **Description:** Compose mounts `./scripts/mongo-init.js`, which does not exist.
- **Risk:** `docker-compose up` fails on launch.  
- **Affected File:** `docker-compose.yml:65`.  
- **Recommendation:** Add the script or remove the bind.

### 7. Default Credentials Committed to Version Control
- **Description:** Compose and env defaults include `admin123`, `redis123`, Grafana admin, and placeholder SMTP secrets.
- **Risk:** Security exposure if copied into any shared environment.  
- **Affected Files:** `docker-compose.yml:67-137`, `.env`.  
- **Recommendation:** Replace with required env overrides and document secret provisioning.

### 8. Service Discovery Not Utilised by Routers
- **Description:** Service discovery subsystem tracks health, but route proxies reference statically configured URLs (`process.env.*`).
- **Risk:** Discovery adds overhead without preventing downtime; manual env drift remains.  
- **Affected Files:** `server/routes/assessments.js:24`, `server/routes/objectives.js:24`, etc.  
- **Recommendation:** Inject service URLs from discovery or unify configuration to avoid dual systems.

### 9. Placeholder Data in Production Paths
- **Description:** Key endpoints still return stubbed responses (e.g., `Demo Business` payload).
- **Risk:** Functional gaps block production readiness and undermine analytics accuracy.  
- **Affected File:** `server/routes/businesses.js:15-23` (others similar).  
- **Recommendation:** Implement real data retrieval before release cut.
