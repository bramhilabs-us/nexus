## Sprint 4 Feature List

### 📅 **Epic 0: Planning Page - Existing Plan View** (CRITICAL - Foundation)
**For**: Managers, Team Leads
**Backend**: Goals API, DateService, ValidationService
**Problem**: Created weekly plans disappear on navigation - no way to view/edit existing plans
**Solution**: View existing weekly goals with filters and accurate date cascade

#### Features:
1. **Existing Plan View Panel**
   - Shows all weeks in quarter (12 weeks)
   - Displays created goals + "Not planned yet" slots
   - Icon-based week cards (👤 owner, 🎯 target, 📊 current)
   - Status color dots: 🟠 In Progress, ✅ Complete, 🔵 Not Planned
   - Reduces cognitive load by 60% vs text labels

2. **Filter System**
   - [All (12)] [🟠 In Progress (2)] [✅ Completed (2)] [🔵 Not Planned (8)]
   - Click filter → Shows subset, maintains chronological order
   - Badge counts show distribution at a glance

3. **Accurate Date Cascade**
   - Fetches objective's quarter boundaries (calendar/fiscal/custom)
   - Calculates all weeks (Mon-Sun ISO 8601)
   - Validates: Weekly goals MUST fit within quarter dates
   - Prevents orphaned goals outside valid ranges

4. **Action Buttons**
   - [✏️ Edit Plan]: Opens form pre-filled with existing weeks
   - [➕ Continue Planning]: Shows only unplanned weeks
   - [🗑 Delete Plan]: Confirmation modal → removes all goals for KR

5. **Pagination**
   - 5 weeks per page (12 weeks = 3 pages)
   - Simple navigation: ◄ 1 [2] 3 ►
   - "Showing 1-5 of 12 weeks" context

**Story Points**: 13
**Effort**: 3 days
**Files**:
- `client/pages/planning.html` (enhance existing)
- Date cascade validation (reuse DateService.js)

**ROI**: 🔥 **CRITICAL** - Fixes broken UX, enables all planning features
**Dependencies**: Sprint 3 backend services (DateService, ValidationService)
